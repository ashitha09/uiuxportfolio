Designing a Smart Cab Booking Experience for GoCab

Project Overview

In today’s fast-paced world, booking a cab should be effortless, intuitive, and efficient. GoCab is a fictional mobile app built for seamless urban transportation. This project focused on designing a smart, sleek, and responsive user interface for a modern ride-hailing platform that prioritizes speed, clarity, and user comfort. The aim was to create an experience that’s familiar yet futuristic—balancing intuitive usability with bold visual cues and subtle animations. The platform caters to both first-time users and seasoned commuters, offering transparency in ride details, easy navigation, and smooth onboarding.

Usability Testing

Methodology: Remote usability testing with 5 participants (ages 23–38, all regular cab app users). Prototype was tested in Figma with scenario-based tasks while observing time-to-complete and user feedback.

Test Scenarios

  • Complete sign-up and log in
  • Book a ride from current location to a set destination
  • View driver details and track the ride in real time
  • End a ride and leave a tip + rating

Key Findings

  • Task Completion: All users completed every scenario successfully.
  • Booking Speed: Average booking time was under 40 seconds after onboarding.
  • Clarity: Participants appreciated the bold CTAs and one-tap “Get a Ride” button.
  • Ride Tracking: Real-time map with ETA was described as “reassuring” and “clearer than Uber.”
  • Tipping & Feedback: The end-ride modal was easy to use, but 2 participants wanted the tip options to be more prominent.
  • Accessibility: One participant suggested larger fonts for older users in driver info cards.

User Feedback

“I booked my ride in seconds—it felt faster than the apps I use daily.”
“The live tracking screen feels clean and easy to follow.”
“The tipping option could stand out a little more at the end.”

Iterations

  • Improved contrast of driver section
  • Streamlined tipping UI
  • Reduced booking steps from 5 to 3
